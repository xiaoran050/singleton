1.单例模式：
    只分配一个内存，节省资源
    避免资源多重占用
    设置全局访问点，严格控制访问
    缺点:
    没有接口，扩展困难
    如果要扩展单例对象，只有修改代码。

单例模式必须有私有构造器
保证线程安全的情况有：饿汉式/多重检查锁懒汉式/静态内部类懒汉式/枚举式
延迟加载：延迟加载，也叫延迟实例化，延迟初始化等，
主要表达的思想就是：把对象的创建延迟到使用的时候创建，而不是对象实例化的时候创建。这种方式避免了性能的浪费。
列如：匿名内部类
防止序列化和反序列化破坏单列：利用枚举单列或者添加
private  Object readResolve(){
     return  INSTANCE;
}
防御反射攻击单例：利用枚举单例。
